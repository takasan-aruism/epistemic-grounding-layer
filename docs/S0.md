# SPEC S0 — 観測ビュー(status_views.py)

task_id: S0-status-views
根拠: roadmap-wiring-v0.1 STAGE 0 / TASK-B 実測(DE-0447)
実装ファイル: status_views.py(これのみ。テストファイルは出力禁止)
合格条件: 同梱 test_status_views.py 全件合格(runner が実測する)

## 目的

2DER の一次記録から観測ビューを**導出**する。書かれるものではなく導出される
もの(handoff 置換の第一歩)。LLM を使わない。純粋な読み取りと整形のみ。

## 制約

```
- 標準ライブラリのみ(外部依存禁止)。
- ファイル書き込みをしない。すべての関数は文字列/データを返すだけ。
- 壊れた行で例外を投げて全体を止めない。数えて報告する(可視化された失敗)。
  ただし設定不備・入力パス不在は ViewError。
- 出力順序は決定的(同じ入力なら同じ出力)。
```

## 実装するもの

```python
class ViewError(Exception): ...
```

### 1. load_runs(runs_root) -> list[dict]

`runs_root` 直下の各 run ディレクトリ(`<run_id>/`)を走査。
`RESULT_PACKET.json` と `run_log.jsonl` を読む。

戻り値の各要素:
```python
{"run_id": str, "task_id": str, "status": str, "iterations_used": int,
 "needs_human": bool, "chain_ok": bool, "chain_detail": str,
 "iterations": [ {"iteration": int, "test_exit_code": int|None,
                  "output_tail": str, "reply_excerpt": str} ],
 "events": int}          # run_log の行数
```
- `chain_ok` は run_log.jsonl の hash chain を**自前で再検証**した結果
  (各行 `prev_hash` の連結・`seq` 連番・`record_hash` の再計算一致)。
  `record_hash` は「record_hash 以外の全フィールドを
  `json.dumps(sort_keys=True, ensure_ascii=False)` した bytes の sha256」。
  不一致なら `chain_ok=False`、`chain_detail` に最初の不一致位置(例 `"line 3: seq"`)。
  検証成功なら `chain_detail="ok"`。
- `iterations` は run_log の `TEST_RUN`(exit_code / output_tail)と
  `MODEL_REPLY`(reply の先頭 200 文字を `reply_excerpt`)を iteration で突合。
  TEST_RUN が無い iteration も MODEL_REPLY があれば含める(test_exit_code=None)。
- RESULT_PACKET.json が壊れている/無い run は結果に含めるが
  `status="UNREADABLE"`, `iterations_used=0`, `needs_human=True` とする。
- 並び順: `run_id` の降順。
- `runs_root` が存在しない場合は空リスト(例外にしない)。

### 2. load_gaps(events_path) -> list[dict]

EGL の `events.jsonl` を読む。1行1イベント、schema:
`{event_id, ts, run_id, event_type, object_type, object_id, payload}`。

`object_type == "KnowledgeGap"` のイベントのみ対象。同一 `object_id` に
複数イベントがある場合、**ファイル出現順で後のものが勝つ**(event sourcing)。
payload をマージ(後勝ち)して現在状態を作る。

戻り値の各要素:
```python
{"gap_id": str, "question": str, "status": str,
 "required_for": list, "is_required": bool}
```
- `is_required` は `required_for` が非空なら True(DF-2 importance の代理指標)。
- `gap_id` は payload の `gap_id`、無ければ `object_id`。
- 並び順: `is_required` True が先、その中で `gap_id` 昇順。
- パス不在なら空リスト。JSON として壊れた行は無視して数える(§5参照)。

### 3. load_backlog(path) -> list[dict]

1行1 JSON。同一 `id` は後勝ち。各要素はそのまま(最低限 `id`/`state`/`title`)。
並び順: state 優先度
`["AWAITING_TAKA","AWAITING_GPT","BLOCKED","IN_PROGRESS","OPEN","DONE"]`
の順、同順位は `id` 昇順。未知 state は最後尾。パス不在なら空リスト。

### 4. load_ledger(path) -> list[dict]

1行1 JSON の DE 台帳。フィールド名が環境で異なりうるため、
`de_id` / `id` / `DE` のいずれか最初に見つかったキーを DE 識別子とする。
どれも無い行は `unparsed` として数え、結果に含めない。

戻り値の各要素: `{"de_id": str, "record": dict}`。
並び順: `de_id` の降順(新しい DE が上)。パス不在なら空リスト。

### 5. derive_metrics(runs, gaps, backlog, ledger, unparsed) -> dict

```python
{"runs_total": int, "runs_passed": int, "runs_needs_human": int,
 "chain_failed": int,
 "gaps_total": int, "gaps_open": int, "gaps_required_open": int,
 "backlog_awaiting": int, "backlog_done": int,
 "ledger_count": int, "unparsed_lines": int}
```
- `runs_passed` は `status == "PASSED"` の数。
- `gaps_open` は `status == "OPEN"`、`gaps_required_open` はさらに
  `is_required` True のもの。
- `unparsed` は呼び出し側が渡す int(ローダーの戻り値はリストのみ)。
  `build` は入力 jsonl(events / backlog / ledger)を走査し、
  JSON として解釈できない行の合計を自分で数えて渡す。

### 6. render_html(metrics, runs, gaps, backlog, ledger, generated_at) -> str

単一 HTML 文字列。以下を**必ず含む**(テストが検査する):
```
- 文字列 "2DER" と generated_at
- 文字列 "derived"(導出物であることの明示)
- 各セクション見出し: "METRICS" / "RUNS" / "GAPS" / "BACKLOG" / "LEDGER"
- 各 run の run_id と status
- chain_ok が False の run には文字列 "CHAIN_FAIL"
- needs_human が True の run には文字列 "NEEDS_HUMAN"
- is_required かつ status=="OPEN" の gap には文字列 "REQUIRED"
- unparsed_lines > 0 のとき文字列 "UNPARSED"
```
HTML エスケープを行うこと(`<` `>` `&` を含むデータで壊れない)。
デザインは自由。CSS はインラインでよい。

### 7. build(config) -> dict

```python
{"html": str, "metrics": dict}
```
`config` は dict。必須キー `runs_root` / `out_title`。
任意キー `events`(events.jsonl パス)/ `backlog` / `ledger`。
必須キーが無い場合 `ViewError`。
`generated_at` は `time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime())` を使う。
上記ローダーを呼び、`unparsed` を合算して `derive_metrics` に渡し、
`render_html` の結果を返す。**ファイルには書かない。**
